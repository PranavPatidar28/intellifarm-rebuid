Intellifarm MVP · Hackathon Story

Farmer-first user flow

This version is designed for demos and judging: the journey reads left to right, the major product moments are color-coded, and the support and admin branches stay visible without overwhelming the main farmer story.

OTP access Fast sign-in for farmers with a simple phone-first entry point.
3-step onboarding Profile, plot, and season setup turn an empty account into a working farm workspace.
Decision support loop Dashboard, planning, market insight, disease help, and grounded assistant all connect back to action.

Presentation Notes

Lead with the main farmer path first: login, onboarding, dashboard, then the action loops. After that, point out the support lane and the admin lane as differentiators.

Legend

  • Access and trust
    Phone OTP and session creation.
  • Onboarding
    Profile, farm plot, and first crop season setup.
  • Daily workspace
    Dashboard, fields, tasks, and recurring farm decisions.
  • Advisory tools
    Planning, support, markets, schemes, and disease help.
  • Admin operations
    Rules, templates, schemes, and report review.
  • Decision points
    Branching moments like auth, onboarding, and role checks.

Hackathon user flow diagram
